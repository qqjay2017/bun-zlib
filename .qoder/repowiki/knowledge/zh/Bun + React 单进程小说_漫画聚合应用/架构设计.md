- 入口 index.ts 使用 `Bun.serve` 在同一进程中同时承担静态页面（`index.html`）与 API 网关：`fetch: router` 把请求交给 TanStack Router，`routes: { "/": index }` 返回前端壳。
- 控制器采用“副作用注册”模式：index.ts 显式 import 四个 controller 文件，每个 controller 在加载时调用 `defineController` 向全局路由表追加 `/api/*` 路由；lib/controller 暴露的 `createRouter()` 由 index.ts 调用并作为 `fetch` 处理器。
- 前端侧 frontend.tsx 作为浏览器入口，挂载 `<RouterProvider>` 和 `<QueryClientProvider>`，路由树由 backend_api 模块中的 `./lib/router` 组装，与后端控制器路径一一对应。
- shared_lib 是前后端共同依赖：后端通过它访问缓存、书源配置与下载逻辑，前端通过它消费同一份路由定义与查询客户端，保证 URL 与数据契约一致。
- backend.ts 提供独立的 WebView 抓取能力（复用本地 Chrome 或自启），供下载流程渲染动态页面后再提取 HTML，与 REST 控制器解耦。
