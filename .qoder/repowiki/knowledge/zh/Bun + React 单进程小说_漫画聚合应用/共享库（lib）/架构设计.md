本目录按职责拆分为若干独立模块，通过明确的依赖方向组织：
- `controller.ts` 实现轻量级全局路由注册表（defineController + createRouter），被后端入口消费以挂载 API；`router.ts` 则基于 @tanstack/react-router 组装前端路由树，两者分别对应服务端与客户端的“路由”概念。
- `cache-manager.ts` 定义 `.cache/<contentType>/<sourceId>_<bookId>/...` 的文件布局，并通过 `save* / load*` 函数暴露统一的元数据与章节缓存接口；`download-manager.ts` 作为唯一消费者，在 executor 流程中先查缓存再调用 `fetchPageHtml → extractor → saveChapter`，并以 `downloads/tasks.json` 持久化任务状态。
- `download-types.ts` 与 `cache-types.ts` 是这两个模块之间的契约，确保缓存写入结构与下载任务字段对齐。
- `source-config.ts` 维护书源注册表，被 `download-manager.ts` 按需拉取 extractors，形成「下载器 → 书源配置」的单向依赖。
- `epub-builder.ts`、`query.ts` 由 core_utilities 子模块统一管理，供上层业务复用。
整体依赖方向：controller/router ← download-manager ← cache-manager ← source-config，类型文件贯穿上下游，避免循环引用。
