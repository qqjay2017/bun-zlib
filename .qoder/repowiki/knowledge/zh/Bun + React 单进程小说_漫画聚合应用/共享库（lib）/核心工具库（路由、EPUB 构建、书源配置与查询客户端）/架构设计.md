本模块由四个相互独立的工具文件组成，均作为 `shared_lib` 下的纯函数/单例导出，供上层业务按需 import：
- `controller.ts`：实现一个类 NestJS 风格的装饰器式路由框架，通过 `defineController(basePath, routes)` 将形如 `"GET /path"` 的键值对注册到全局 `routeRegistry`，再由 `createRouter()` 冻结注册表并返回兼容 Bun.serve 的 `(req) => Response` 处理器，内置路径参数解析（`:param`）、路径标准化与统一 500/404 错误处理。
- `epub-builder.ts`：基于 `jszip` 与 `Bun.file`，按 EPUB 3.0 规范生成 `container.xml`、`content.opf`、`toc.ncx`、`nav.xhtml` 与各章节 XHTML，并通过魔数检测封面图片 MIME；对外暴露 `buildEpub(options)` 与 `buildEpubToFile(options, path)` 两个异步入口。
- `source-config.ts`：维护内部 `Map<string, BookSourceConfig>` 注册表，提供 `registerSource`、`getSourceById`、`getSourceByDomain`、`getAllSources(contentType?)` 四类访问 API，用于在 WebView 中执行 DOM 提取器的书源插件机制。
- `query.ts`：仅导出一个使用 `@tanstack/react-query` 创建的 `QueryClient` 单例，集中配置 1 分钟 staleTime 与 1 次重试。
依赖方向单向向外：各文件之间无互相引用，仅依赖外部包（jszip、@tanstack/react-query）与运行时（Bun、crypto）。
